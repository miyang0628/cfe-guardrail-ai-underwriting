## Overview

This repository contains the complete reproducible codebase for the paper:

> *"Adverse Selection Risk and Counterfactual Explanation Guardrails  
> in Healthcare AI Underwriting: A Quantitative Governance Evaluation  
> under Korea's AI Basic Act"*  
> *(Manuscript under review)*

We propose a quantitative framework for identifying and mitigating
**adverse selection risk** arising from counterfactual explanation (CFE)-based
recourse in AI-assisted health insurance underwriting.  
Two novel metrics are introduced:

- **Reliable Recourse Rate (RRR)** — proportion of declined applicants
  with at least one actionable, constraint-compliant recourse pathway
- **Recourse Accessibility Gap (RAG)** — demographic equity disparity
  in recourse access across age groups

A **three-stage guardrail framework** (Feature Governance → 
Constraint-Embedded CFE → Operational Monitoring) is evaluated against
Korea's AI Basic Act governance dimensions (G1 Transparency,
G2 Robustness, G3 Accountability, G4 Fairness).

## Data Access

This study uses the **Korea National Health and Nutrition Examination Survey
(KNHANES) 2020–2024**, conducted by the Korea Disease Control and Prevention
Agency (KDCA).

- **Access:** [https://knhanes.kdca.go.kr](https://knhanes.kdca.go.kr)
- **Format:** SAS7BDAT files (`hn20_all.sas7bdat` … `hn24_all.sas7bdat`)
- **Availability:** Publicly available upon registration (free of charge)
- **Placement:** Download and place files in `data/` before running notebooks

```
data/
├── hn20_all.sas7bdat
├── hn21_all.sas7bdat
├── hn22_all.sas7bdat
├── hn23_all.sas7bdat
└── hn24_all.sas7bdat
```

## Reproduction Instructions

Run notebooks **in the following order**:

```bash
# Step 1 — Data preprocessing
jupyter nbconvert --to notebook --execute \
    notebooks/01_data_preprocessing.ipynb

# Step 2 — Underwriting model training
jupyter nbconvert --to notebook --execute \
    notebooks/02_ml_underwriting.ipynb

# Step 3 — CFE generation + baseline RRR/RAG
jupyter nbconvert --to notebook --execute \
    notebooks/03_cfe_rrr.ipynb

# Step 4 — Guardrail framework + post-guardrail RRR/RAG
jupyter nbconvert --to notebook --execute \
    notebooks/04_guardrail.ipynb

# Steps 5–10 — Supporting analyses (order-independent)
jupyter nbconvert --to notebook --execute \
    notebooks/06_descriptive_stats.ipynb
jupyter nbconvert --to notebook --execute \
    notebooks/07_model_comparison.ipynb
jupyter nbconvert --to notebook --execute \
    notebooks/08_calibration.ipynb

## Ethical Statement

- KNHANES data are de-identified and publicly available.
  No individual-level linkage or re-identification was performed.
- No additional ethics approval was required for secondary analysis
  of publicly available de-identified survey data,
  consistent with KDCA data use guidelines.
- This study uses KNHANES solely as a **proof-of-concept vehicle**;
  results should not be interpreted as benchmarks for deployed
  underwriting systems.

## Reproducibility Notes

- All random seeds are fixed at `RANDOM_SEED = 42`
  throughout the pipeline.
- Optuna studies use `TPESampler(seed=42)`.
- Train/test splits use `random_state=42`.
- CFE generation uses `random_seed=42` in DiCE calls.
- Minor numerical differences ($< 0.5\%$) may arise
  due to platform-specific floating-point behaviour
  or package version differences.
